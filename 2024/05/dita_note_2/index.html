Dita_note_2

Good sentences in technical documentation identify who is doing what to whom.

An imperative verb is a command. Many items in numbered lists start with imperative verbs. Sentences that start with an imperative verb imply an actor. The implied actor is you.

Use the active voice most of the time. Use the passive voice sparingly. Be bold—be active.

In technical writing, clarity takes precedence over all other rules.

To engage and educate readers, choose precise, strong, specific verbs. Reduce imprecise, weak, or generic verbs, like forms of be (is, are, am, was, were) (sometimes), occur and happen.

Three ways to reduce there is/there are:

  1. Simply delete There is or There are (and possibly another word or two later in the sentence).

  2. Repair a There is or There are sentence by moving the true subject and true verb from the end of the sentence to the beginning.

  3. If no subject exists, consider creating one.

Minimize certain adjectives and adverbs–Feed your technical readers factual data instead of marketing speak. Refactor amorphous adverbs and adjectives into objective numerical information. For example:

Setting this flag makes the application run screamingly fast.

Setting this flag makes the application run 225-250% faster.
